# キャラクター画像スタンプ生成スキル

このスキルは、ユーザーが「（キャラクター名）の画像を作って」と要求した際に、nanobanana（画像生成・編集ツール `generate_image` など）を用いて、特定のルールで画像編集を行うための指示書です。

## 1. 入力の解析
- ユーザーの要求から「キャラクター名」を抽出します。
- 要求が「（キャラクター名）の画像を作ってp」や「〜作ってh」のように、「作って」のうしろにアルファベットなど特定の文字（p, h, j など任意）の指定があるかどうかを確認します。

## 2. 対象ファイルの特定
以下のパスから、対象の元画像とスタンプ画像を取得します。
- **元画像フォルダ**: `/Users/yamamotokenta/Library/CloudStorage/GoogleDrive-kenta.bishi777@gmail.com/マイドライブ/キャラ画像/{キャラクター名}/`
  （※フォルダ内にある `original` という名前の画像ファイルを元画像として選択します）
- **スタンプ画像**: `/Users/yamamotokenta/Library/CloudStorage/GoogleDrive-kenta.bishi777@gmail.com/マイドライブ/キャラ画像/アクセサリ/stamps.jpg`

## 3. 画像編集（nanobanana）の実行
エージェントの持つ画像生成・編集ツール（またはnanobanana相当のAI編集機能）を使い、元画像とスタンプ画像をツールに渡して以下の編集を依頼します。

1. **微細な変形**: 
   - 画像の傾きを ±0.5〜1度 程度、色調（明るさ、コントラスト）を ±1〜2% 程度変化させる。（元の画像と並べてようやく気づく程度のわずかなランダム変化）
2. **スタンプの合成**:
   キャラクター名に応じて以下の分岐処理を行います。
   - **yukko または erika の場合**:
     - スタンプの合成処理は一切行わない（色調・傾きの微細な変形処理のみを適用する）。
   - **iori の場合**:
     - 元画像（original）にピンクのハートが3つ配置されているため、選んだ1種類のスタンプを3箇所すべてに配置し、それぞれのハートが完全に見えなくなるように置き換え（上書き）する。
   - **上記以外のキャラクターの場合**:
     - 顔の「鼻のみ」「口のみ」「鼻と口両方」の3パターンのうち1つをランダムに選ぶ。
     - 元画像（original）に既に口や鼻を隠す別のスタンプが適用されている場合、元のスタンプが確実に見えなくなるように完全に置き換え（上書き）する。
     - 選んだ部位を隠すように、スタンプ画像を自動配置（リサイズ・合成）する。（サイズ感は、該当のパーツが隠れる程度・パーツの1.2倍程度のお任せ比率）スタンプは一つだけ使う。

## 4. 保存と報告
- 編集が完了したら、新しい画像を指定されたキャラフォルダ内に保存します。
- **ファイル名ルール**: 保存時のファイル名は実行日の和暦日付に基づく数字5桁とします。フォーマットは「令和の年(1桁)+月(2桁)+日(2桁)」です（例: 令和8年4月6日の場合は `80406`）。
- **プレフィックスの追加**: ユーザーの要求の末尾にアルファベット（p, h, jなど）が指定されていた場合は、保存時のファイル名の先頭にそのアルファベットを付与します。（例: p指定なら `p80406`、h指定なら `h80406` となります。設定後、必要に応じて拡張子を付けます）
- 処理が完了したこと、保存したファイルのパスをユーザーに報告します。
